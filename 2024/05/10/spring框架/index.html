@RequestMapping(“/admin”)
在Spring框架中，特别是Spring MVC，@RequestMapping是一个用来映射HTTP请求到控制器处理方法的注解。当你在一个类或方法上使用@RequestMapping注解时，你定义了一个URL模式，Spring容器将根据这个模式将请求路由到相应的处理器方法。
这个注解会将所有标记了/admin的URL路径映射到这个类中的处理方法上。也就是说，任何以/admin开始的URL路径都将由这个类中的某个方法来处理。（映射到admin上）

@RequestMapping(value = “/reg”, method = RequestMethod.GET)
value = “/reg”：定义了URL映射，意味着当一个GET请求的URL路径是/reg时，Spring MVC会调用这个方法。
method = RequestMethod.GET：指定了HTTP请求方法为GET，这表示只有当HTTP请求类型为GET时，这个方法才会被触发。

@PostMapping 是 @RequestMapping 注解的一个特化形式，专门用于处理 HTTP POST 请求。POST 请求通常用于向服务器提交数据，这些数据会包含在请求体中

final：表示一旦变量被初始化后，就不能再重新赋值。

@Autowired 可标注在构造方法上，setter方法上或者是@Bean方法上

@RestController 和 @RequestMapping ：用于创建RESTful web服务
@RestController = @Controller + @ResponseBody
@RestController 它提供JSON，XML和自定义响应

@RequestMappping 用于定义访问REST端点的Request URL。可以定义Request方法来使用和生成对象。默认请求方法是:GET。
