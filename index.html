@RequestMapping(“/admin”)
在Spring框架中，特别是Spring MVC，@RequestMapping是一个用来映射HTTP请求到控制器处理方法的注解。当你在一个类或方法上使用@RequestMapping注解时，你定义了一个URL模式，Spring容器将根据这个模式将请求路由到相应的处理器方法。
这个注解会将所有标记了/admin的URL路径映射到这个类中的处理方法上。也就是说，任何以/admin开始的URL路径都将由这个类中的某个方法来处理。（映射到admin上）

@RequestMapping(value = “/reg”, method = RequestMethod.GET)
value = “/reg”：定义了URL映射，意味着当一个GET请求的URL路径是/reg时，Spring MVC会调用这个方法。
method = RequestMethod.GET：指定了HTTP请求方法为GET，这表示只有当HTTP请求类型为GET时，这个方法才会被触发。

@PostMapping 是 @RequestMapping 注解的一个特化形式，专门用于处理 HTTP POST 请求。POST 请求通常用于向服务器提交数据，这些数据会包含在请求体中

final：表示一旦变量被初始化后，就不能再重新赋值。

@Autowired 可标注在构造方法上，setter方法上或者是@Bean方法上

@RestController 和 @RequestMapping ：用于创建RESTful web服务
@RestController = @Controller + @ResponseBody
@RestController 它提供JSON，XML和自定义响应

@RequestMappping 用于定义访问REST端点的Request URL。可以定义Request方法来使用和生成对象。默认请求方法是:GET。

linux常见命令

  1. ls - 列出目录内容

    • ls: 列出当前目录下的文件和文件夹
    • ls -l: 以长格式列出详细信息
    • ls -a: 显示所有文件，包括隐藏文件
    • ls -R: 递归列出所有子目录的内容

  2. cd - 改变当前目录

    • cd: 切换到用户的主目录
    • cd ~: 同上
    • cd -: 切换到上一个工作目录
    • cd /path/to/dir: 切换到指定的目录

  3. pwd - 打印当前工作目录

    • pwd: 显示当前所在的完整路径

  4. cp - 复制文件或目录

    • cp source destination: 将源文件复制到目标位置
    • cp -r source destination: 递归复制目录及其内容

  5. mv - 移动或重命名文件或目录

    • mv old_name new_name: 重命名文件或目录
    • mv source destination: 将文件或目录移动到目标位置

  6. rm - 删除文件或目录

    • rm file: 删除文件
    • rm -r directory: 递归删除目录及其内容
    • rm -f file: 强制删除文件，不提示确认

  7. mkdir - 创建新目录

    • mkdir directory: 创建指定的目录
    • mkdir -p /path/to/dir: 创建多级目录结构

  8. rmdir - 删除空目录

    • rmdir directory: 删除空目录

  9. touch - 创建空文件

    • touch file: 创建一个空文件，如果文件已存在则更新其时间戳

  10. chmod - 更改文件或目录的权限

    • chmod 644 file: 设置文件的权限，所有者具有读写权限，组和其他用户具有读权限
    • chmod -R 755 /path/to/dir: 递归设置目录及其内容的权限，所有者具有读写执行权限，组和其他用户具有读执行权限

  11. chown - 更改文件或目录的所有者

    • chown user:group file: 更改文件的所有者和组
    • chown -R user:group /path/to/dir: 递归更改目录及其内容的所有者和组

  12. find - 在目录树中搜索文件

    • find / -name file: 在整个文件系统中搜索名为 file 的文件
    • find /path/to/dir -type d: 在指定目录及其子目录中搜索目录
    • find /path/to/dir -type f: 在指定目录及其子目录中搜索文件

  13. grep - 在文件中搜索模式

    • grep 'pattern' file: 在文件中搜索包含 pattern 的行
    • grep -r 'pattern' /path/to/dir: 递归在目录中搜索包含 pattern 的文件

  14. sed - 流编辑器，用于处理文本

    • sed 's/old/new/g' file: 将文件中的 old 替换为 new
    • sed -n '3,5p' file: 打印文件的第 3 行到第 5 行

  15. awk - 模式扫描和处理语言

    • awk '{print $1, $3}' file: 打印文件的每一行的第一列和第三列
    • awk '/pattern/ {print $0}' file: 打印包含 pattern 的行

  16. tar - 归档文件

    • tar -cvf archive.tar /path/to/dir: 创建一个包含指定目录的 tar 归档文件
    • tar -xvf archive.tar: 解压 tar 归档文件

  17. gzip - 使用 Lempel-Ziv 编码压缩文件

    • gzip file: 压缩文件
    • gzip -d file.gz: 解压 gzip 压缩的文件

  18. bzip2 - 使用 bzip2 算法压缩文件

    • bzip2 file: 压缩文件
    • bzip2 -d file.bz2: 解压 bzip2 压缩的文件

  19. du - 检查磁盘使用情况

    • du -sh /path/to/dir: 显示指定目录的总磁盘使用情况

  20. df - 报告文件系统的磁盘空间使用情况

    • df -h: 以易读的格式显示磁盘空间使用情况

  21. ps - 显示当前进程状态

    • ps aux: 显示所有进程及其详细信息
    • ps -p PID: 显示指定进程的信息

  22. top - 实时显示和管理系统资源

    • top: 显示实时更新的进程和资源使用情况

  23. kill - 发送信号给进程

    • kill PID: 发送 SIGTERM 信号给指定的进程
    • kill -9 PID: 发送 SIGKILL 信号强制终止进程

  24. ln - 创建链接（硬链接或软链接）

    • ln -s /path/to/source /path/to/link: 创建指向源的软链接

  25. uname - 显示系统信息

    • uname -a: 显示所有系统信息

  26. history - 显示用户命令历史

    • history: 显示用户执行过的命令历史

前缀和及差分

前缀和基本定义：按照索引对前n个数依此进行相加，得到一个新的数组

例：
输入一个长度为n的整数序列。接下来再输入m个询问，每个询问输入一对l, r。对于每个询问，输出原序列中从第l个数到第r个数的和。
思路：很容易想到用暴力去写完这个题目，但是思考一下，有没有更好的解决办法？
我们可以先做一个预处理，定义一个sum[]数组，sum[i]代表a数组中前i个数的和

查询操作：

由此，我们引出差分数组，它是前缀和的逆运算

首先给定一个原数组a：a[1], a[2], a[3],,,,,, a[n];
然后我们构造一个数组b ： b[1], b[2], b[3],,,,,, b[i];
使得 a[i] = b[1] + b[2] + b[3] + ,,,,,, + b[i]

构造差分:

是不是和前缀和很像a[n]就相当于sum[n],b[n]=sum[n]-sum[n-1],被我找出端倪来了吧哈哈哈哈哈哈哈
所以这有什么用呢？接下来让我们来看看

*给定区间[l, r ]*，让我们把a数组中的[l, r] 区间中的每一个数都加上c,即 a[l] + c , a[l + 1] + c , a[l + 2] + c ,,,,,, a[r] + c;

暴力可能是我们最先想到的，万物皆可暴力（bushi）

另一种思路：
首先让差分b数组中的 b[l] + c ,通过前缀和运算，a数组变成 a[l] + c ,a[l + 1] + c,,,,,, a[n] + c;
然后我们打个补丁，b[r + 1] - c, 通过前缀和运算，a数组变成 a[r + 1] - c,a[r + 2] - c,,,,,,,a[n] - c;

相当于，前r个数的前缀和加c,然后再减去前l个数加c,所以就是a[r]+c-(a[l]-c),用b数组来表示就是
a[r]+c=b[r]+a[r-1]-c
a[l]-c=b[l]-a[l-1]+c
