MySQL登录时出现Access denied for user ‘root‘@‘localhost‘ (using password: YES)无法打开的解决方法

1.首先检查项目文件目录下中的resoures中的.yml配置文件，看是不是username和password错误，和自己的mysql的账户和密码是否一致，不一致需要改动。
2.用 --init-file 参数在服务启动时加载并运行修改密码的命令文件，该命令一旦执行，服务启动后密码即已经清除或者重置，启动服务后即可以空密码或指定密码登入。(参考csdn,没用到这一个方法)

step 1:先关掉mysql服务器 net stop mysql

要打开管理员模式，在搜索中搜cmd,然后右键用管理员模式打开

step 2:创建一个 txt 文本文件，命名为 mysqlc(即mysqlc.txt)，然后在文本里编写一条修改密码的命令。我是在安装MySQL的目录下创建的。

在里面写好修改的密码，例：ALTER USER 'root'@'localhost' IDENTIFIED BY '123456'

step3:命令行方式启动服务器，指定启动时执行上述的密码修改命令文件。(先要确认mysql是关闭状态)

mysqld --init-file=D:\mysql\mysqlc.txt --console

step4:接着重启服务器

net start mysql

step4:然后用刚刚改过的密码输入就可以了
